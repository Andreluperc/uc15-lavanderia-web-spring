# Evidências de Testes e Bugtracking

Sugestão: use o GitHub Issues (ou Trello/Jira, se preferir).

## Modelo de registro de falha (Issue)
- Título: [BUG] <resumo>
- Ambiente: Windows / navegador / Java 17
- Passos para reproduzir:
  1) ...
  2) ...
- Resultado esperado:
- Resultado obtido:
- Severidade: Baixa / Média / Alta
- Evidência: print do erro
- Status: Aberto / Em correção / Resolvido
- Commit de correção: <hash/print>

## Lista de testes executados (exemplos)
- CT-01 Cadastrar Cliente
- CT-02 Listar Clientes
- CT-03 Cadastrar Operador
- CT-04 Login CPF+Matrícula (sucesso)
- CT-05 Login inválido (falha esperada)
